# OpenClaw 批量巡检开发任务清单

## 1. 开发目标

把“企业微信里 `@小智` 发起批量巡检”做成可落地能力，核心闭环是：

- 企业微信消息里拿到发送人和被 `@` 人信息
- 按规则解析本次批量巡检归属负责人
- 用负责人 `managerWecomUserId` 去后台匹配门店
- 结合巡检计划过滤出真正可执行门店
- 自动创建批次并执行
- 把进度和完成结果回到企业微信群
- 后台批量任务中心能查看来源、范围、进度和失败重试

当前开发进度：

- [x] 线程 A 的负责人上下文提取、负责人解析、企业微信批量创建已接通
- [x] 线程 B 的批量请求体、负责人字段、批次查询接口已补齐
- [x] 线程 C 的批量任务中心展示、来源/负责人来源展示已补齐
- [ ] 企业微信群进度播报
- [ ] 企业微信群完成汇总播报
- [ ] 批次完成后自动同步企业微信表格 / 巡检记录

## 2. 线程拆分

建议拆成 3 条开发线程并行推进：

### 线程 A：企业微信消息入口线程

目标：

- 拿到企业微信消息里的发送人、`@` 人、会话信息
- 新增负责人解析逻辑
- 在对话入口里支持“按负责人门店发起批量巡检”

负责内容：

- 企业微信事件结构梳理
- `fromUserId` / `mentionedUserIds` 注入
- 批量话术解析
- `resolveBatchOwner()` 负责人归属判定
- 批量任务预回复 / 进度播报 / 完成播报

建议写入范围：

- `scripts/upgrade-wecom-inspection-scene2.js`
- `scripts/patch-wecom-plugin.js`
- 如果插件代码目录可控，则进入对应消息处理模块

交付标准：

- 企业微信群里 `@小智 执行巡检计划 我名下门店 营业画面巡检`
- 能正确识别发送人
- 能生成面向发送人负责门店的批量巡检任务

### 线程 B：巡检后端线程

目标：

- 把负责人范围的批量巡检做成标准后端能力
- 为企业微信入口提供稳定的批量创建和进度查询支撑

负责内容：

- 批量请求体字段补齐
- 负责人范围门店筛选
- 负责人来源字段沉淀
- 批次汇总、详情、重试接口增强
- 批量进度播报数据结构准备

建议写入范围：

- `inspection-api/cmd/server/main.go`

交付标准：

- 后端能直接根据 `managerWecomUserId + planId`
- 返回预估结果
- 创建批次
- 查询批次进度
- 支持失败重试

### 线程 C：后台产品配套线程

目标：

- 让后台能看出这个批量任务是不是企业微信负责人发起的
- 让运营和排障人员能看懂来源、范围、负责人和状态

负责内容：

- 批量任务中心展示增强
- 批次来源、负责人来源展示
- 筛选和文案优化
- 命中日志与批量任务视角联动

建议写入范围：

- `inspection-admin/src/views/BatchCenterView.vue`
- `inspection-admin/src/api/inspection.ts`
- `inspection-admin/src/types/inspection.ts`
- `inspection-admin/src/utils/inspection.ts`

交付标准：

- 后台能看出本批次是：
  - 后台手动创建
  - 企业微信负责人批量创建
- 能看出负责人来源：
  - 文本指定
  - `@` 指定
  - 发送人默认
