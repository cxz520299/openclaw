# OpenClaw 批量巡检执行清单

## 1. 目标

把批量巡检从“后台可创建”推进到“企业微信里 `@小智` 后，系统能自动识别发起人或被 `@` 负责人，并去后台匹配这个负责人绑定的门店，再自动创建批量巡检任务”的闭环能力。

目标链路：

- 企业微信群里 `@小智`
- OpenClaw 拿到消息体中的人员信息
- 识别本次批量巡检要按谁的门店范围执行
- 去后台按 `managerWecomUserId` 匹配负责人
- 找到这个负责人名下绑定的门店
- 结合巡检计划筛出可执行门店和监控模块
- 创建批次任务
- 执行中持续反馈进度
- 完成后回企业微信群播报汇总结果
- 同步写入企业微信表格 / 巡检记录

---

## 2. 闭环定义

这次要补齐的不是普通“批量任务中心”，而是下面这条企业微信闭环：

1. 用户在企业微信群发送指令并 `@小智`
2. 消息里要么显式带负责人，要么默认使用发消息的人
3. OpenClaw 从企业微信事件里拿到：
   - 发送人 `from_user_id`
   - 被 `@` 用户列表 `mentioned_user_list`
   - 群聊 ID / 会话 ID
   - 原始文本
4. 系统判断本次批量范围归属人：
   - 优先取指令里明确指定的负责人
   - 其次取消息里被 `@` 的负责人
   - 最后兜底取消息发送人
5. 用归属人的 `wecom_user_id` 去后台门店表匹配：
   - `stores.managerWecomUserId`
6. 拉出该负责人名下全部启用门店
7. 再按巡检计划过滤掉没有绑定计划的门店
8. 自动生成批次
9. 回企业微信预回复：
   - 已识别负责人
   - 已命中门店数
   - 已跳过门店数
   - 预计耗时
10. 执行完成后回企业微信最终汇总

## 4. 核心规则

## 4.1 批量范围归属人判定规则

建议统一规则：

1. 如果消息正文里明确说了负责人姓名或工号，优先用正文命中
2. 如果正文没说，但消息里 `@` 了某个负责人，取第一个非机器人 `@` 用户
3. 如果正文和 `@` 都没有负责人，则默认取消息发送人
4. 如果归属人的 `wecom_user_id` 在后台没有匹配到负责人门店，则返回引导文案，不创建批次

建议不要直接把所有 `@` 人都当成巡检范围，避免多人群聊误触发。

## 4.2 门店匹配规则

负责人范围门店获取规则：

- 从门店表中查询：
  - `status = enabled`
  - `managerWecomUserId = 归属人 wecom_user_id`
- 如果门店数为 0：
  - 返回“该负责人当前未绑定门店”
- 如果门店数 > 0：
  - 继续按巡检计划绑定关系过滤

## 4.3 计划过滤规则

从负责人门店集合里继续筛：

- 只保留已经绑定当前巡检计划的门店
- 没绑定计划的门店进入 `skippedStores`
- 预回复里要明确告诉用户：
  - 选中多少家
  - 实际命中多少家
  - 跳过多少家

## 10. 最终要达到的业务体验

业务在企业微信群里只需要说：

- `@小智 执行巡检计划 我名下门店 营业画面巡检`
- `@小智 执行巡检计划 @张三名下门店 营业画面巡检`

系统就能自动完成：

- 识别本次批量巡检归属负责人
- 去后台匹配这个负责人绑定的门店
- 过滤出真正绑定了该计划的门店
- 自动创建批次并执行
- 在群里持续反馈进度
- 完成后给出汇总结果
- 同步沉淀到企业微信表格

这条链路跑通以后，批量巡检才算真正从“后台能力”升级到“企业微信可直接用的产品能力”。
